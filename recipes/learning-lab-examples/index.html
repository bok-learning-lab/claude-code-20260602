Faculty Recipes for Claude Code & Cowork

Four worked examples of how instructors and course staff can use an LLM harness — Claude Code or Cowork — to take real teaching and research tasks off their plate. Each is a small, self-contained project: source files go in inputs/, the prompt or command lives in operations-tools-commands/, and the finished artifact lands in outputs/.

A reference set for workshops · drawn largely from GENED 1104 “Science & Cooking”

The shared pattern

Every example follows the same shape: gather the real materials you already have (a syllabus, past exams, papers, a recording), tell Claude who it is and what “good” looks like, and let it produce a draft you review. The point isn’t automation — it’s a knowledgeable first pass that keeps your judgment in the loop.

1 · Resolving a Schedule Conflict

class-schedule-conflict/

Course planning Trade-off analysis One-shot prompt

“My guest lecturer from Flour Bakery can only come on the 30th — but something’s already scheduled that day. What do I do?”

Claude is given the full course schedule and syllabus, then asked to find the conflict, weigh 2–3 concrete fixes against their pedagogical trade-offs (does moving a session break a topic sequence?), recommend one, and draft a revised schedule in the original format.

Inputs course schedule + syllabus (Markdown)
Operation a single planning prompt
Outputs revised-schedule.md / .html

Why it’s a good example: shows Claude reasoning about teaching priorities, not just shuffling calendar cells — and producing a usable, formatted artifact in one pass.

2 · Generating a Makeup Exam

makeup-exams/

Assessment Reads PDFs & DOCX CLAUDE.md persona Grading rubric

“A student missed Midterm 2. I need a fresh exam at the same difficulty — plus an answer key my TFs can grade from.”

Claude reads several years of past midterms and keys to calibrate difficulty, format, and the course’s signature culinary framing, then writes a brand-new exam (no reused questions) and a separate answer key with partial-credit rubrics. A CLAUDE.md casts it as a TA who knows the course and must err toward the harder side.

Inputs syllabus + a folder of past exams/keys
Operation generate exam, then key, in separate passes
Outputs makeup-exam + answer-key (.md / .html)

Why it’s a good example: calibration from real artifacts, a project-level persona via CLAUDE.md, and a deliberate separation of exam from key so answers don’t leak into the questions.

3 · Research Summaries with a Pedagogical Twist

research_helper/

Research synthesis Batch processing Self-contained HTML Linked index

“Turn this stack of arXiv papers into readable summaries — and tell me, honestly, what each one means for using LLMs in teaching and research.”

For each paper Claude writes a faithful neutral summary (claims, method, evidence, limitations) and then, as a distinct pass, “the twist”: an honest bridge to the research agenda — explicitly saying “this doesn’t transfer” when that’s the truthful reading. Every file is portable HTML, and an index.html hub links them all.

Inputs arXiv PDFs + markdown extracts
Operation summarize each, build a linking index
Outputs one HTML per paper + index.html hub

Why it’s a good example: faithfulness over relevance — Claude is instructed to mark speculation and resist overstating, exactly the discipline scholarly synthesis needs.

4 · “Top Tips” from a Workshop Recording

day-1-summary/

Transcription Shell + API Distillation Audience-aware writing

“We recorded Day 1 of the workshop. Pull out the durable advice — not a play-by-play of the demos.”

A bundled shell script chunks the recording and transcribes it with diarization in parallel; Claude then reads the noisy transcript and distills ~10–15 principles for non-engineer faculty, using demos only as illustration and preserving every caveat about data sensitivity faithfully.

Inputs diarized session transcript
Operation transcribe (script) → distill tips (prompt)
Outputs day-1-top-tips.md / .html

Why it’s a good example: pairs a real tool (a transcription script) with careful prompt design — extract the mental models, not the minutes — and writes for a specific, non-technical audience.

Each folder runs on its own. Open its operations-tools-commands/ file for the exact prompt or command, drop your materials into inputs/, and review what Claude writes to outputs/. The throughline across all four: keep human judgment in the loop — the harness extends the instructor, it doesn’t replace them.
