Workshop Projects

Seven small projects, all built on the same shape: raw material in, a defined operation in the middle, a finished artifact out.

The pattern: inputs → operations → outputs

inputs/→ operations-tools-commands/→ outputs/

Every folder here repeats one structure. inputs/ holds the raw material you start with — a dataset, a stack of papers, a transcript, a syllabus. operations-tools-commands/ holds the operation: the prompt, script, or command that transforms the input — and often a CLAUDE.md that tells Claude Code how to do the work. outputs/ holds the artifact that comes out the other end.

The point of separating these is repeatability. When the operation is written down rather than improvised in a chat, you can rerun it on new inputs, hand it to someone else, or change one stage without disturbing the others. That discipline — deciding deliberately what goes into the work — is context engineering, and it is the single habit these projects are meant to teach.

Why so many outputs are HTML

Several projects end in an .html file rather than a plain document, and that choice is deliberate. Markdown is perfect when the result is text to read. HTML is the right output when the result needs visual structure (cards, tables, side-by-side columns, a styled index), interaction (links between pages, filters, a live chart you can manipulate), and to be self-contained and shareable — an .html file opens in any browser with no install, so you can email it to a colleague or hand it to students as-is.

You don't write the HTML by hand. You describe what the reader should be able to see and do, and the harness produces the files. It widens what an artifact can be: not a flat document, but a small, styled, navigable thing — which is exactly what the population pyramids, the recipe site, the research summaries, and the glossary below are.

The projects

01 Close Reading

Shakespeare's complete works → close-reading prompt → an analytical essay

The input is a large literary corpus: the complete works of Shakespeare as plain text. The operation applies a close-reading prompt that directs Claude to do sustained, evidence-cited analysis of a chosen passage rather than a summary. The output is a single long-form analytical essay in Markdown.

close-reading-output.md markdown

02 Population Pyramids

UN WPP 2024 population CSVs → charting prompt → interactive demographic charts

The input is structured tabular data — UN World Population Prospects 2024 CSVs of population by country, age, and sex. A charting prompt turns the raw numbers into a small program that reads the data and renders an age–sex pyramid comparing two countries. The output is a pair of self-contained, interactive HTML charts you manipulate in the browser.

chart 1 chart 2 html · no index

03 Recipes

Family recipes → formatting prompt → a browsable recipe site

The input is a set of informal family recipes in mixed, inconsistent plain text. A formatting prompt normalizes each one into a consistent structure — ingredients, steps, yield — and styles the collection as a set. The output is a browsable HTML site with an index page linking every recipe.

index.html html site

04 Class Schedule Conflict

GENED 1104 schedule + syllabus → conflict-resolution prompt → a revised schedule

The input is two course documents — the GENED 1104 schedule and syllabus — that contain a scheduling clash. The operation reasons over both to locate the conflict and propose a minimally disruptive fix that respects topic order and assessment dates. The output is a revised schedule in Markdown.

revised-schedule.md markdown

05 Makeup Exams

Syllabus + past Midterm 2 materials → exam-generation prompt → a new exam + answer key

The input is course material: the syllabus plus several years of past Midterm 2 exams and answer keys. An exam-generation prompt calibrates difficulty and format to the originals and writes new, non-duplicated questions in a pass kept separate from the key. The output is two Markdown files — a fresh makeup exam and its complete answer key with grading rubrics.

makeup-exam.md answer-key.md markdown

06 Research Helper

arXiv long-context papers → summary prompt → structured research summaries

The input is a set of arXiv long-context research papers (PDFs plus one markdown extract). A summary prompt produces a faithful neutral summary of each, then adds an honest "twist" connecting the findings to teaching and research with LLM harnesses. The output is an HTML site — one structured summary per paper plus an index, with key figures embedded for the Context Rot report.

index.html html site

07 Summary of Day 1

Video + transcript → transcribe + still-grab scripts → step handouts + a jargon glossary

The input is the recording of the first workshop session — the video and its diarized, time-stamped transcript. Transcription and still-grab scripts plus a doc-writing prompt reorganize the live session into step-by-step instructions and define the vocabulary that was new to participants. The output is an HTML resource set: two setup handouts and a linked jargon glossary.

index.html html site
Each project also has a CLAUDE.md (where present) describing how to work in it. New to the vocabulary? Start at the glossary.
