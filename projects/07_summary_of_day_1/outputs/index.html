Day 1 — Resources

Turning the first workshop session into materials participants can actually keep.

This project takes the recording of Day 1 — the video and its diarized, time-stamped transcript — and turns a three-hour live session into a small set of durable resources: handouts that summarize the steps we covered, a distilled set of the instructors' pro-tips, and a glossary of the jargon and new terminology that came up. The aim is that someone who attended (or missed) the session can follow what we did without rewatching the video.

video + transcript→ transcribe & still-grab scripts→ handouts + glossary

The raw transcript lives in inputs/; the scripts and the doc-writing prompt that produced these resources live in operations-tools-commands/; everything below is the outputs/. The two setup guides reorganize the spoken session into do-this-next steps; the glossary defines every term that was new to participants, so a guide can link to a definition instead of re-explaining it.

Handouts — the steps we covered

Claude.ai Setup Guide

The browser interface on the HUIT plan: confirming your account, the interface tour, models, memory, attachments, and a worked artifact example (the interactive population pyramid).

Claude Co-Working Setup Guide

Co-working as a workflow: the inputs → operation → outputs loop, writing real prompts, and setting up a project that remembers context across sessions.

Pro-tips — how to think about the tools

Top Tips for Using LLMs and Harnesses

The guidance the instructors kept returning to — which surface to use (claude.ai vs. Cowork vs. Code), write-the-code-instead, give-it-context, the context window, new-chat hygiene, model choice, data sensitivity, and keeping your judgment in the loop.

Glossary — the new vocabulary

Workshop Glossary

Plain-language definitions of every term that was new to participants — Claude, model, context, context rot, prompt, artifact, HTML, harness, and more — each a standalone page the handouts link into.
